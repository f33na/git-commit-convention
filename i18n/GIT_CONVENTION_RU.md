# Стандарт коммитов Git

> Версия стандарта: 1.0.0

---

## Формат коммита

```
[TASK-ID/TAG] Краткое описание изменения
```

| Часть | Обязательность | Описание |
|---|---|---|
| `TASK-ID` | Опционально | Идентификатор задачи (совпадает с именем ветки) |
| `TAG` | Обязательно | Тег типа изменения (см. таблицу ниже) |
| Описание | Обязательно | Краткое описание на языке проекта, до 72 символов |

---

## Режимы использования

### Полный формат — есть задача и ветка

Используется когда работа ведётся в отдельной ветке, привязанной к задаче.  
Имя ветки совпадает с `TASK-ID`.

```
[TASK-ID/TAG] Описание
```

```bash
# Примеры
[AUTH-42/FEATURE] Добавить OAuth2 авторизацию через Google
[PAY-7/FIX] Исправить округление суммы при конвертации валют
[API-15/MAJOR] Изменить структуру ответа /users endpoint
[UI-3/REFACTOR] Вынести кнопки в отдельный компонент Button
```

---

### Упрощённый формат — нет задачи или ведётся малый проект

Используется когда:
- проект небольшой и задачи не формализованы
- нет системы трекинга задач
- один разработчик
- коммит идёт напрямую в `main` или `dev`

```
[TAG] Описание
```

```bash
# Примеры
[INIT] Инициализация проекта
[FEATURE] Добавить тёмную тему
[FIX] Убрать утечку памяти в WebSocket handler
[CHORE] Обновить зависимости
[RELEASE] v1.3.0
```

---

## Теги изменений

| Тег | Описание | Влияние на версию |
|---|---|---|
| `INIT` | Инициализация проекта или модуля | — |
| `FEATURE` | Новая функциональность | **MINOR** ↑ |
| `FIX` | Исправление бага | **PATCH** ↑ |
| `HOTFIX` | Срочное исправление на продакшене | **PATCH** ↑ |
| `MAJOR` | Ломающее изменение, несовместимость API | **MAJOR** ↑ |
| `REFACTOR` | Рефакторинг без изменения поведения | **PATCH** ↑ |
| `PERF` | Улучшение производительности | **PATCH** / **MINOR** ↑ |
| `STYLE` | Форматирование, линтинг, отступы | — |
| `DOCS` | Документация | — |
| `TEST` | Добавление или правка тестов | — |
| `CHORE` | Рутина: зависимости, конфиги, CI/CD | — |
| `REVERT` | Откат предыдущего коммита | **PATCH** ↑ |
| `RELEASE` | Релизный коммит, простановка версии | — |

---

## Семантическое версионирование (SemVer)

Формат версии: **`MAJOR.MINOR.PATCH`**

| Уровень | Когда увеличивается |
|---|---|
| **MAJOR** | Ломающие изменения (`MAJOR`) |
| **MINOR** | Новая функциональность (`FEATURE`, `PERF` при значимом улучшении) |
| **PATCH** | Исправления и мелкие правки (`FIX`, `HOTFIX`, `REFACTOR`, `REVERT`, `PERF`) |
| **—** | Версия не меняется (`STYLE`, `DOCS`, `TEST`, `CHORE`, `INIT`, `RELEASE`) |

### Правило определения следующей версии

Смотришь все коммиты с момента последнего `RELEASE`:

```
Есть хотя бы один MAJOR?           → MAJOR++,  MINOR = 0, PATCH = 0
Есть хотя бы один FEATURE / PERF*? → MINOR++,  PATCH = 0
Всё остальное (FIX, HOTFIX, …)?    → PATCH++
Только STYLE / DOCS / TEST / CHORE → версия не меняется
```

> *`PERF` повышает `MINOR` только если изменение значимо для пользователя.

## Релизный коммит и Git-тег

```bash
# Коммит-маркер релиза
git commit --allow-empty -m "[RELEASE] v1.3.0"

# Git-тег
git tag -a v1.3.0 -m "Release v1.3.0"
git push origin v1.3.0
```

---

## Правила оформления описания

- Длина — не более **72 символов**
- Используй **повелительное наклонение**: *Добавить*, *Исправить*, *Удалить*
- Не ставь точку в конце
- Пиши на языке, принятом в команде (фиксируется в этом файле)
- Если нужен подробный контекст — добавь тело коммита через пустую строку:

```
[AUTH-42/FEATURE] Добавить OAuth2 авторизацию через Google

Реализована авторизация через Google OAuth2.
Добавлены переменные окружения GOOGLE_CLIENT_ID и GOOGLE_CLIENT_SECRET.
Затронуты модули: auth, config, middleware.
```

---

## Именование веток

Имя ветки должно совпадать с `TASK-ID` из коммита.

```
AUTH-42     ← ветка для задачи AUTH-42
PAY-7       ← ветка для задачи PAY-7
```

Постоянные ветки (`main`, `dev`) не имеют `TASK-ID` — для них используется упрощённый формат коммита.

---

## Быстрая шпаргалка

```
Есть ветка/задача?
  ДА  →  [TASK-ID/TAG] Описание
  НЕТ →  [TAG] Описание

TAG влияет на версию?
  MAJOR   → x+1.0.0
  FEATURE → x.y+1.0
  FIX     → x.y.z+1
  прочие  → без изменений
```
